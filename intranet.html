Does web browsers support opening .application or .exe links on Intranet pages?

Aqua examples:

   - .application links (click-once Microsoft apps): Engineering Index, APD, RP, CSDashboard, Warranty Claims, APD Schedule, etc.

   - .exe links: Epicor, Engineering Index

Microsoft Edge (Browser)

1. .application (ClickOnce) links:

   - No, ClickOnce links are blocked by default for security reasons.

2. .exe links:

   - No, .exe links are blocked by default. Browsers do this for security reasons (to prevent drive-by execution of malware).

Google Chrome (Browser)

1. .application (ClickOnce) links:

   - No, ClickOnce links are blocked by default for security reasons.

2. .exe links:

   - No, .exe links are blocked by default. Browsers do this for security reasons (to prevent drive-by execution of malware).

Aqua Examples:
  • Engineering Index (.exe link)

Browser errors when attempting to click above links on our Intranet site:


Why this is happening?
Allowing a webpage to reach into the local filesystem (or file shares) and run executables would open Aqua for multiple security attacks, so modern browsers disallow these completely.
